## Liteverse 0.4.0 — Developer Preview

Liteverse 0.4.0 refines the literature universe into a calmer, more readable
research workspace and moves deterministic preparation work into a short-lived
native helper. Scientific interpretation, evidence decisions, and research
memory remain explicit Codex responsibilities.

### What changed

- **A more focused interface.** The public and private builds now share one
  Apple-inspired dark design system with system typography, restrained
  materials, consistent spacing, clearer drawers, and responsive navigation.
- **Cleaner universe rendering.** Region labels, paper heat, relationship
  beams, hit targets, focus transitions, and narrow-window layouts were refined
  without changing verified/candidate relationship semantics.
- **Lower idle cost.** Canvas work is split into cached layers, image resources
  are reused, pointer hit testing is frame-limited, and rendering scales from
  30 fps while interacting to 12 fps while idle and 4 fps in the background.
- **Native local preparation.** The bundled `LiteverseLocalWorker` performs
  deterministic hashing, explicit arXiv retrieval, PDF validation, page-by-page
  extraction, strict duplicate detection, and draft-only review preparation.
  It runs on demand and exits when the job finishes.
- **Clear AI boundaries.** Local preparation and Context Preview never create
  scientific evidence, alter Usage, promote a relationship, or replace the
  Curator, Retriever, and Research Memory Skills.
- **Safer workspace writes.** Cross-process locks, revision-pinned adoption,
  atomic JSONL writes, transaction recovery, stricter backup validation, and
  fail-closed error handling protect the graph and project memory.
- **Deterministic layout support.** Local layout localization and up-to-ten
  broad nebula regions preserve the existing background-aware, lower-overlap
  placement contract.
- **Safer public/private synchronization.** Shared code moves through a narrow
  allowlist, while papers, PDFs, graph state, research memory, annotations, and
  Usage data remain private.

### Verification

- Public-release privacy and English-source checks passed.
- Lint, TypeScript validation, the macOS package build, and all 78 tests passed.
- All three bundled Liteverse Skills passed the Skill Creator validator.
- The 1,000-card benchmark measured a 21.6 ms warm-search p95 and a 112.2 ms
  Context Pack build on the release machine.
- The packaged seed contains zero regions, zero papers, zero relations, and no
  private paper cards.
- The app and bundled native Worker passed arm64 and code-signature checks.

### Install

This preview requires macOS 13 or later on an Apple Silicon Mac. Unzip the
macOS archive, move `Liteverse.app` to Applications, then Control-click it and
choose **Open** on first launch.

The downloadable app is ad-hoc signed and is not notarized by Apple. It remains
a developer preview rather than a production-signed release.

### Current boundaries

Liteverse 0.4.0 has no account, cloud sync, background daemon, bundled language
model, automatic search for unrelated literature, update feed, or shipped stdio
MCP adapter. The public build starts with an empty local library.
