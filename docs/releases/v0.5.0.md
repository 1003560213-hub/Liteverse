## Liteverse 0.5.0 — Developer Preview

Liteverse 0.5.0 expands the literature universe into a navigable hierarchy and
moves more deterministic preparation work into the local app. Scientific
interpretation, evidence decisions, and research-memory meaning remain explicit
AI-assisted review tasks.

### What changed

- **Nebulae, galaxies, and paper stars.** Broad research themes remain nebulae,
  each nebula can contain up to twelve focused galaxies, and individual papers
  appear as stars inside a selected galaxy.
- **Galaxy relationship lanes.** The universe now summarizes paper-level
  evidence as parallel verified or candidate lanes between galaxies. The
  underlying paper relations and evidence records remain intact.
- **Research-memory black holes.** Every nebula has a central black hole for
  project notes and knowledge cards. Empty ones clearly report `No Notes`.
- **A calmer, more legible interface.** Refined system typography, neutral
  materials, consistent controls, keyboard navigation, bounded labels, and
  responsive drawers preserve the cinematic universe without visual clutter.
- **More local intake choices.** Users can add a PDF or arXiv link, link an
  existing literature folder without duplicating its PDFs, or use the read-only
  Zotero intake path for stored PDF attachments.
- **Less routine AI work.** The native Worker now handles page extraction,
  metadata anchors, strict SHA-256/arXiv/DOI duplicate checks, local indexing,
  incremental classification candidates, and three-to-five-paper Review
  Packets before scientific review.
- **Transactional batch adoption.** Multiple reviewed batches can be adopted
  safely without re-extracting unchanged sources. Revision pins, stale-lock
  recovery, fail-closed catalog guards, and resumable transactions protect the
  graph from partial updates.
- **Scalable deterministic layout.** Galaxies form stable three-dimensional
  rings around the memory black hole, reuse artwork fairly, preserve nebula
  bounds, and keep rendering work bounded as libraries grow.
- **Public/private separation.** The public app starts empty. Shared code,
  tests, Skills, and visual assets are synchronized without copying private
  papers, graph state, project memory, annotations, Usage, or local paths.

### Verification

- Public privacy, empty-seed, English-source, lint, TypeScript, and data checks
  pass before packaging.
- The full automated suite covers hierarchy layout, keyboard navigation,
  linked sources, Zotero intake, local preparation, batch review and adoption,
  research memory, crash recovery, and performance guards.
- A 1,000-item queue test confirms that heavy PDF and manifest validation is
  limited to the active three-to-five-paper batch.
- The packaged public seed contains zero regions, zero papers, zero relations,
  and no private paper cards.
- The macOS app and short-lived native Worker are verified as arm64 binaries
  with valid ad-hoc code signatures.

### Install

This preview requires macOS 13 or later on an Apple Silicon Mac. Unzip the
macOS archive, move `Liteverse.app` to Applications, then Control-click it and
choose **Open** on first launch.

The downloadable app is ad-hoc signed and is not notarized by Apple. It remains
a developer preview rather than a production-signed release.

### Current boundaries

Liteverse does not silently turn extracted text or numeric similarity into a
verified scientific relationship. Exact evidence and scientific meaning still
require review. Scanned PDFs without a usable text layer are marked for OCR,
but this version does not bundle OCR. Linked folders must remain available at
their registered locations. There is no account, cloud sync, background daemon,
bundled language model, update feed, or automatic download of unrelated papers.
